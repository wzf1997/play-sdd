状态机 & 数据流

useScratchCardGame hook 内部状态转换

IDLE
初始态，等待 init 返回
→
GET /api/game/scratch-card/init
← { cards[9], sessionId }
↓
PRIZE_REVEAL
奖池展示 800ms，动画播放
setTimeout 800ms 后自动进入下一态
↓
GATHERING
物理引擎运行，卡片聚拢
useCardPhysics → 所有卡片到达中心后触发
↓
SPREADING
卡片散开展开到九宫格位置
各卡片到达目标位置后 → WAITING
↓
WAITING
等待用户选中一张卡
→
用户点击某张卡 → selectedCardIndex 存入 state
↓
SCRATCHING
ScratchCanvas 激活，用户手刮
→
刮除率 ≥ 65%
POST /api/game/scratch-card/play
← { prize }
↓
REVEALED
奖品展示 + WinModal 弹出
倒计时从后端 expireAt 计算，点击跳转/唤起 App
防作弊设计
奖品由后端在 init 时已决定并存入 scratch_sessions，前端只在刮完后发 play 请求获取结果，无法提前知晓。
