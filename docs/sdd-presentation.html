Core Concept

什么是 SDD？

Spec-Driven Development 是一种以规格文档为核心的 AI 协作开发范式。

传统开发依赖开发者经验与直觉，而 SDD 强调：先写规格，再生成代码。规格是 AI 的行为边界，也是团队协作的共识基础。 
# 传统方式
idea → code → review → bugs → fix...

# SDD 方式
idea
  → /brainstorming       # 需求明确
  → Pencil MCP           # UI先行
  → /opsx:propose        # 规格生成
  → /writing-plans       # 任务分解
  → /using-git-worktrees # 隔离开发
  → /subagent-driven-dev # AI执行
  → /requesting-review   # 规格审查
  → /browser_visible     # 可视验证
  → /finishing-a-branch  # 完成合并
  → /opsx:archive        # 归档变更
  → /capture-knowledge   # 知识沉淀
  → ✓ 可审计的高质量交付物

proposal.md — Why
现有方案缺乏 跨游戏可复用的 Landing 基础设施。每次新游戏上线都需重写相似的首页逻辑，缺少统一的设计语言约束和插件化能力，导致代码重复、维护成本高。
8 大能力规格
1 多游戏皮肤系统 — CSS token 热切换
2 插件化游戏注册 — GamePlugin interface
3 BFF 数据聚合层 — API 路由代理
4 SEO 基础设施 — 结构化数据 + 元信息
5 A/B 测试支持 — 实验配置注入
6 性能预算 — LCP < 2.5s, CLS < 0.1
7 测试策略 — Unit + Integration + E2E
8 Monorepo 结构 — pnpm + Turborepo

K001 · SDD 最佳实践
规格先行比代码先行降低 70% 返工率
在本次实践中，由于提前定义了 contract.md 和 schema.ts，AI 生成的代码 API 接口零偏差，仅在非规格层面发现 7 个质量问题
K002 · 架构模式
GamePlugin 插件模式适用于多品牌 Landing
将游戏特定逻辑封装为 GamePlugin，骨架层只依赖接口而非实现，新游戏上线只需实现 plugin 即可，接入成本 < 2小时
K003 · 工具链洞察
Pencil MCP → Design Token → CSS 是最短路径
UI 先行 + Token 导出 + 代码直接消费，消除了设计稿与实现的视觉偏差，无需反复对齐

// GamePlugin Interface (contract.md)
interface GamePlugin {
  name:    string
  version: string
  meta: {
    title:       string
    description: string
    ogImage:     string
    keywords:    string[]
  }
  tokens: SkinTokens   // CSS token map
  sections: SectionConfig[]
  register(registry: GameRegistry): void
}

// 注册新游戏
export const gameAlpha: GamePlugin = {
  name: 'game-alpha',
  version: '1.0.0',
  meta: { title: 'Game Alpha', ... },
  tokens: alphaTokens,
  sections: [heroSection, featuresSection],
  register(r) { r.add(this) }
}
新游戏接入 3 步流程
1
实现 GamePlugin 接口，定义皮肤 tokens 和页面 sections
2
调用 registry.add(plugin) 完成注册
3
BFF 层自动代理数据，骨架层渲染完整 Landing 页

Core Values

SDD 的 6 大核心价值

01
规格即文档
规格文件 IS 文档。写一次，AI 执行、人工审阅、团队共识三用，消除文档与代码的分裂
02
可审计性
每个实现决策都可追溯到对应的规格条目。代码为什么这样写？因为 contract.md 第 12 行这样规定
03
AI 友好
结构化规格是 AI 最好的输入。有了规格约束，AI 生成的代码接口零偏差，大幅降低返工率
04
可复用性
骨架层与皮肤层分离，新游戏只需实现 GamePlugin 接口即可复用整套 Landing 基础设施
05
UI 先行
Pencil MCP 先出设计稿，导出 Design Tokens，代码直接消费。消除视觉偏差，无需反复对齐
06
知识留存
/capture-knowledge 将每次项目实践中的洞察结构化，沉淀到团队知识库，形成组织资产
The End
